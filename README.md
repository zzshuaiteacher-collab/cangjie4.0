> *「先确认，再搬运，在 Obsidian 长出 99% 仿生的灵魂」*

**仓颉 v4.1 是目前最深度的团队风格蒸馏方案。** 它不仅融合了「女娲」的调研能力，更集成了 **「WeChatArchiver」** 的微信全量无损搬运能力，通过 Phase 0 的强制目标确认，杜绝无效检索。

---

## 🔥 v4.1 核心升级：

1.  **强制目标报告 (Phase 0)**：在启动前生成背景简报供用户确认，确保「找对人」。
2.  **集成 WeChatArchiver**：针对微信公众号执行「地毯式无损入库」，保留图片、评论和排版原貌。
3.  **9 路并行 Agent**：调研深度全面对齐并超越「女娲」，覆盖从商业决策到情感温度的所有维度。
4.  **路径合规保护**：强制在用户指定的 Obsidian 路径内运行，严禁存放在 Agent 临时目录。

---

## 🛠️ 运行流程：

### 1. 提交目标
用户输入：`「仓颉，蒸馏 [团队名]」`

### 2. 审阅计划书 (Phase 0)
AI 会先给出一份简报，包括该团队在公众号、视频号、全网的资源量评估。
> **用户需回复「执行」方可继续。**

### 3. 全量搬运 (Phase 1)
AI 启动 9 路 Agent，调用 `WeChatArchiver` 将所有文章按年份无损搬运至您的 Obsidian。

### 4. 知识入库与重构 (Phase 2)
基于搬运的全文，AI 自动构建实体 Wiki、隐喻仓库和思想演化图谱。

### 5. 仿生写作与进化 (Phase 3-4)
基于本地「全真大脑」，产出具有高度思想一致性的深度内容。

---

## 📂 配套 Skill 说明
本仓库 Skill 需要配合以下环境：
- **WeChatArchiver.md**：位于 `F:\Skill集合\微信搬运skill`，负责公众号的全文抓取。
- **Obsidian**：建议用户预先安装并提供一个空白仓库路径。
